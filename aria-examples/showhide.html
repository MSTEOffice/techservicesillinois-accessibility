Accessibility Examples: ARIA Show/Hide

Show/Hide regions are collapsible regions in a page that are controlled (i.e. expanded or collapsed) by a button or a link. Show/Hide regions are relatively simply to implement accessibly, requiring very little JavaScript.

Five ARIA attributes are needed to implement a Show/Hide region:

  • aria-controls (property) - Establishes a programmatic relationship between the region toggle and the region.
  • aria-expanded (state) - True if the region is expanded; false if the region is collapsed.
  • aria-hidden (state) - False if the region is expanded; true if the region is collapsed.
  • aria-labelledby (optional state) - Optionally labels the region. Necessary if role=region is used on the region.
  • region (optional role) - Optionally defines the collapsible region as a distinct region.

The markup for the show/hide is as follows for the collapsed state:

<button id="hs-toggle" class="btn" aria-controls="hs-rgn" aria-expanded="true">Toggle Region 1</button>
<div id="hs-rgn" role="region" aria-labelledy="hs-toggle" aria-hidden="false"> … </div>

When expanded, the values of the aria-expanded and aria-hidden states are reversed:

<button id="hs-toggle" class="btn" aria-controls="hs-rgn" aria-expanded="false">Toggle Region 1</button>
<div id="hs-rgn" role="region" aria-labelledy="hs-toggle" aria-hidden="true"> … </div>

JavaScript is necessary to update the aria states for the toggle and region. CSS should be used to update the visual state of the show/hide region. If a CSS attribute selector is used, there is no need to add or remove any CSS classes:

[aria-hidden=true] {
   display: none;
}

Show/Hide Region with Toggle Button

This region was initially hidden. A screen reader user will hear Toggle Region 1 button expanded, or Toggle Region 1 button collapsed when focus is placed on the toggle button. The region will be identified as Toggle Region 1 region.

When using a link as the toggle for a show/hide region, it is necessary to add role=button to the link. This is because the link is being used as a button, not for navigation. The href attribute can either be empty ('#') or javascrip:void(); may be used. The code would look something like this:

<a id="hs-toggle" href="javascript:void();" aria-controls="hs-rgn" aria-expanded="true" role="button">Toggle Region 1</a>
<div id="hs-rgn" role="region" aria-labelledy="hs-toggle" aria-hidden="false"> … <div>
